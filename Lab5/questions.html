Ответы на контрольные вопросы по Flexbox и CSS

1. Какие значения может принимать свойство position и как они влияют на положение элемента?

static (По умолчанию)
Элемент следует обычному потоку документа. Свойства top, right, bottom, left на него не действуют.
relative
Элемент позиционируется относительно своего нормального положения в потоке. Свойства top, right, bottom, left сдвигают его от этого места, не влияя на положение других элементов.
absolute
Элемент "вырывается" из обычного потока. Он позиционируется относительно ближайшего позиционированного (не static) родителя. Если такого родителя нет, он позиционируется относительно <body> (или <html>).
fixed
Элемент "вырывается" из потока и позиционируется относительно окна браузера (viewport). Он остается на месте даже при прокрутке страницы.
sticky
Гибрид relative и fixed. Элемент ведет себя как relative, пока не достигнет определенной точки при прокрутке (заданной top, bottom и т.д.), после чего он "прилипает" к экрану, как fixed.

2. В чем разница между display: block, display: inline, display: inline-block и display: none?

  • display: block: Элемент занимает всю доступную ширину и всегда начинается с новой строки (например, <div>, <p>). Ему можно задавать width, height, margin и padding.
  • display: inline: Элемент занимает только необходимое ему пространство и не начинается с новой строки (например, <span>, <a>). Ему нельзя задать width и height, а вертикальные margin и padding не будут влиять на окружающие элементы.
  • display: inline-block: Сочетает свойства обоих. Элемент ведет себя как inline (выстраивается в строку с другими), но ему можно задавать width, height, margin и padding, как block.
  • display: none: Элемент полностью удаляется со страницы. Он не только не виден, но и не занимает места.

3. Опишите блочную модель CSS. Какие основные свойства влияют на размер и расположение?

Блочная модель CSS (Box Model) описывает, как элементы представлены в виде прямоугольных "коробок". Она состоит из четырех компонентов, идущих изнутри наружу:

  1. Content (Содержимое): Область, где отображается текст или изображения. Ее размеры задаются width и height.
  2. Padding (Внутренний отступ): Прозрачное пространство внутри рамки, между содержимым и рамкой.
  3. Border (Рамка): Линия, окружающая padding и content.
  4. Margin (Внешний отступ): Прозрачное пространство снаружи рамки, отодвигающее элемент от других элементов.

По умолчанию (при box-sizing: content-box), фактическая ширина элемента на экране = width + padding-left + padding-right + border-left + border-right.

4. Как работает свойство flex-direction в контейнере с display: flex?

Свойство flex-direction определяет главную ось flex-контейнера, то есть направление, в котором будут выстраиваться дочерние элементы.

  • row (по умолчанию): Главная ось горизонтальная, слева направо.
  • row-reverse: Главная ось горизонтальная, справа налево.
  • column: Главная ось вертикальная, сверху вниз.
  • column-reverse: Главная ось вертикальная, снизу вверх.

5. Как с помощью align-items и justify-content управлять расположением элементов?

justify-content (Выравнивание по Главной оси)
Управляет выравниванием элементов вдоль главной оси (направление которой задает flex-direction).
  • flex-start: Прижимает элементы к началу оси.
  • center: Группирует элементы в центре оси.
  • space-between: Равномерно распределяет элементы, при этом первый и последний прижаты к краям.
  • space-evenly: Равномерно распределяет элементы, при этом пространство между ними и по краям одинаковое.
align-items (Выравнивание по Поперечной оси)
Управляет выравниванием элементов вдоль поперечной оси (перпендикулярной главной оси).
  • stretch (по умолчанию): Растягивает элементы на всю доступную высоту/ширину контейнера.
  • flex-start: Прижимает элементы к началу поперечной оси.
  • center: Выравнивает элементы по центру поперечной оси.
  • baseline: Выравнивает элементы по их базовой линии (линии текста).

6. Найдете в сети красивое применение flexbox.

Классическими примерами применения Flexbox, которые используются повсеместно, являются:

  • Центрирование: Вертикальное и горизонтальное центрирование любого элемента внутри родителя.
  • Навигационная панель: Создание адаптивной шапки (header), где логотип и элементы меню равномерно распределены.
  • Адаптивные карточки: Создание галерей или блоков контента, которые автоматически перестраиваются в один ряд на больших экранах и в колонку на маленьких (как в предыдущем дополнительном задании).
